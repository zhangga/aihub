# 作者动态（search_author_posts）结果处理与展示规范

## 响应结构体说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `items` | array | 发文列表，每条包含标题、链接、发布时间、内容、作者名和相关性得分 |
| `items[].title` | string | 贴文标题 |
| `items[].url` | string | 原文链接 |
| `items[].publish_time` | string | ISO8601 格式的发布时间，如 `2026-03-07T12:34:56Z` |
| `items[].content` | string | 正文内容，服务端已截断到约 1000 字 |
| `items[].user_name` | string | 作者名 |
| `items[].score` | double | 相关性得分，保留 4 位小数 |
| `authors` | array | 作者信息列表 |
| `authors[].name` | string | 作者名称 |
| `authors[].uid` | string | 作者唯一标识符（禁止向用户展示） |
| `selected_author_name` | string | 当前选择的作者名称 |
| `page_num` | integer | 当前页码 |
| `total_count` | integer | 符合条件的总记录数 |
| `BaseResp` | object | 基础响应，`StatusCode` 为 0 表示成功 |

## 注意事项

- 从用户的查询中提取 `author_name` 字段信息时，**严格按照用户输入的名称进行查询**，不允许进行任何形式的扩展或改写。例如，若用户查询"央视"，则需要严格按照"央视"进行查询，**不能**将其扩展为"央视新闻"或"央视新闻网"等；
- 返回的 `uid` 字段内容仅用于系统内部数据交互，**禁止以任何形式、任何场景向用户展示 `uid` 字段内容**；
- 当检测到 `selected_author_name` 与用户查询名称不完全一致时，**必须向用户展示类似"库中未找到与您查询完全匹配的用户，已返回最相似的结果"的提示信息**；反之，若 `selected_author_name` 与用户查询名称完全匹配，则无需展示提示信息；
- 对于返回数据中的 `selected_author_name` 字段，必须采用视觉突出方式进行展示（建议使用加粗字体），并在其旁添加明确的文本说明，内容为"当前查询内容的作者"；
- 当 `selected_author_name` 与用户查询名称不一致时，必须在界面中以**清晰的列表形式**完整展示返回数据中 `authors` 数组内的所有 `author_name` 信息；
- 作者信息展示区域必须位于回复内容的最开头部分，**包括未完全匹配时的备选作者信息**。完成作者信息展示后，再展示贴文总数，最后展示贴文内容；
- 当用户基于展示的 `author_name` 进行后续提问时，自动再次调用本接口，**同时将用户选择的 `author_name` 相对应的 `uid` 参数作为 `mp_uid` 传递**；
